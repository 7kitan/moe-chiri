Writing a CLI in Rust

2026.06.20

rustclicode

I recently wrote a CLI tool called Route for scaffolding projects. Here is a rough sketch of how it works — argument parsing, file generation, and error handling.

Argument Parsing

Using clap with derive macros keeps things declarative:

use clap::Parser;

#[derive(Parser)]
#[command(name = "route", about = "Scaffold a project")]
struct Cli {
    /// Project name
    name: String,

    /// Template to use
    #[arg(short, long, default_value = "default")]
    template: String,

    /// Output directory
    #[arg(short, long, default_value = ".")]
    out: PathBuf,
}

Generating Files

Walking a template directory and rendering each file with basic variable substitution:

fn scaffold(tpl: &Path, out: &Path, name: &str) -> Result<()> {
    for entry in WalkDir::new(tpl) {
        let entry = entry?;
        let rel = entry.path().strip_prefix(tpl)?;
        let dest = out.join(rel);

        if entry.file_type().is_dir() {
            fs::create_dir_all(&dest)?;
            continue;
        }

        let mut content = fs::read_to_string(entry.path())?;
        content = content.replace("{{name}}", name);
        content = content.replace("{{NAME}}", &name.to_uppercase());

        fs::write(dest, content)?;
    }
    Ok(())
}

Error Handling

Rather than boxing errors everywhere, a custom error enum keeps things clean:

#[derive(Debug, thiserror::Error)]
pub enum Error {
    #[error("template not found: {0}")]
    TemplateNotFound(PathBuf),
    #[error("io error: {0}")]
    Io(#[from] std::io::Error),
    #[error("walk error: {0}")]
    Walk(#[from] walkdir::Error),
}

type Result<T> = std::result::Result<T, Error>;
